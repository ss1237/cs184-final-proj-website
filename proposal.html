Report Proposal Milestone

Shapeshift: Topology-Aware 3D Mesh Interpolation

Saathvik Selvan, Mehdi Khfifi, Julia Isaac, Harbani Jaggi

CS 184, Computer Graphics and Imaging

Summary

For our final project, we propose Shapeshift, a system for interpolating between two 3D triangle meshes with differing topologies. Using the halfedge data structure, we will resample the source mesh to match the target's topology, then perform linear vertex interpolation to morph between the shapes. We may also explore shading and texture interpolation for smoother visual transitions.


Problem Description


Our project, Shapeshift, tackles the challenge of interpolating between two 3D triangle meshes that differ in both geometry and topology. Inspired by visual effects in films such as the Harry Potter series — where characters transform seamlessly from one creature to another—we aim to simulate a similar mesh transformation process in a technically grounded way. Traditional mesh morphing techniques assume the source and target meshes share identical vertex counts and connectivity, but real-world models often do not meet this requirement. Our goal is to develop a pipeline that enables smooth transformations between arbitrary triangular meshes by aligning their topology and interpolating vertex positions.

We begin by representing both meshes using the halfedge data structure, which allows efficient traversal and local connectivity updates. To enable interpolation, we first resample the source mesh so that it matches the target mesh in terms of vertex count and face structure. This may involve either subdivision (upsampling) or simplification (downsampling), depending on which mesh is denser. We aim to preserve surface features as much as possible during this process to avoid introducing geometric artifacts.

Once the meshes have matching topologies, we compute a mapping between corresponding vertices. In the simplest case, this mapping will be based on vertex order, but we will explore proximity-based or feature-preserving matching to ensure higher-quality morphs. We then perform linear interpolation of vertex positions over a sequence of frames, producing a smooth animation from the source to the target mesh.

As optional extensions, we will consider interpolating vertex normals to preserve shading continuity during the transformation. We may also implement per-frame shading using the Phong model and experiment with interpolating texture coordinates if the models are textured. These additions will help enhance the visual realism of the morphing process.


Goals and Deliverables

Baseline Goals

Our baseline objective is to implement a complete mesh interpolation pipeline for two 3D triangle meshes with different topologies. This includes:

  • Representing input meshes using the halfedge data structure.
  • Performing upsampling or downsampling to ensure the source and target meshes have the same vertex count and topology.
  • Computing vertex correspondences and performing linear interpolation of vertex positions across time.
  • Generating a smooth morphing animation between the two meshes.

We aim to produce at least one clean morph sequence between two distinct meshes by the final project deadline.

Extension Goals

If time permits, we plan to explore several extensions to improve visual quality and add technical depth:

  • Interpolation of vertex normals to maintain smooth shading during the transformation.
  • Implementation of dynamic per-frame shading using the Phong model.
  • Interpolation of texture coordinates if the source and target meshes are textured.
  • Incorporation of more advanced vertex matching techniques to reduce artifacts while morphing.

Resources

We will be using the mesh editing and rendering framework provided in the CS 184 starter code as the foundation for our project. This includes support for the halfedge data structure, mesh visualization, and basic interaction tools.

For additional mesh models, we plan to use public 3D geometry datasets such as those available from Sketchfab and Stanford's 3D Scanning Repository. We will also use tools like Blender to manually inspect or preprocess meshes if necessary.

We will refer to research papers and online resources on traditional mesh morphing techniques to guide our implementation, including:

  • Mesh Morphing - Eurographics State-of-the-Art Report (2001)
  • 3D Mesh Morphing - Mocanu, 2013
  • A Comparison of Mesh Morphing Methods - Staten et al., 2009

All implementation will be done locally in C++ using OpenGL for rendering, with collaboration and version control managed via GitHub.
